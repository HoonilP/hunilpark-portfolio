<decisions>
## Implementation Decisions

### 페이지 레이아웃 & 구조
- 히어로 스타일 헤더 — 프로젝트 제목 + 한줄 설명 + 기술 배지 + 대표 이미지(플레이스홀더)가 크게 보이는 상단 영역
- 사이드바 메타 + 본문 구성 — 데스크탑에서 왼쪽(또는 오른쪽) 사이드바에 메타데이터, 오른쪽에 서술 콘텐츠
- 사이드바 풀 메타데이터 — 기술 스택, 역할, 팀 규모, 기간, GitHub/외부 링크 모두 표시
- 모바일 사이드바 처리는 Claude 재량

### 기술 서술 깊이
- 서술 포맷: 한국 대기업/중견기업이 선호하는 프론트엔드 포트폴리오 형식을 리서치하여 Claude 재량으로 결정
- 서술 분량: 위 리서치 결과를 참조하여 Claude 재량으로 결정
- 코드 스니펫 포함 여부: Claude 재량 (필요 시 선택적 포함)
- 프론트엔드 기술 강조점: 프로젝트별로 Claude가 적절히 구성

### 내비게이션 & 전환
- 이전/다음 프로젝트 네비게이션 — 페이지 하단에 이전/다음 프로젝트 이동 버튼
- 브레드크럼 — 상단에 홈 > 프로젝트 > [프로젝트명] 형태의 경로 표시
- 부드러운 전환 애니메이션 — 페이지 전환 시 페이드인 또는 슬라이드 효과
- URL 구조: ID 기반 (`/projects/1`, `/projects/2`, `/projects/3`)

### 미디어 & 비주얼
- 스크린샷: 플레이스홀더로 자리 확보 (기본 회색 박스 스타일)
- 아키텍처 다이어그램: 플레이스홀더로 자리 확보 (기본 회색 박스 스타일)
- 히어로 영역에 프로젝트 대표 이미지 플레이스홀더 포함

### Claude's Discretion
- 모바일에서 사이드바 레이아웃 처리 방식
- 기술 서술 포맷 (한국 대기업 선호 포트폴리오 리서치 기반으로 결정)
- 기술 서술 분량 (리서치 기반으로 결정)
- 코드 스니펫 포함 여부 및 범위
- 프로젝트별 프론트엔드 기술 강조점 구성
- 전환 애니메이션의 구체적 구현 방식

</decisions>
